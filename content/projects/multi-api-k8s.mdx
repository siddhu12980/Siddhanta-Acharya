Three Node.js microservices deployed to Amazon EKS: an auth service that
handles password hashing and JWT issuance, a users service for signup and
login, and a tasks service for CRUD operations protected by JWT. The
infrastructure is defined in Kubernetes manifests using EFS-backed
persistent storage and AWS load balancers for external access.

## What's interesting

The auth service is deliberately stateless — it has no database. Its only
jobs are hashing passwords with bcrypt and signing/verifying JWTs. Both
the users and tasks services call it over HTTP via Kubernetes internal DNS
(`auth-service.default:3000`), which resolves to the ClusterIP without
leaving the cluster. This makes the auth service a shared primitive: any
new service in the cluster can verify tokens by hitting the same endpoint,
with no shared secret distribution problem.

The users service writes a signup log to an Amazon EFS (Elastic File
System) volume mounted at `/app/users/users-log.txt`. Because EFS supports
`ReadWriteMany`, both user-service replicas share the same file — something
you cannot do with a standard EBS volume. This is a small but concrete
demonstration of when shared file storage is the right primitive over a
database.

The Kubernetes manifests wire up three deployments (1 auth replica, 2
each for users and tasks), two external load balancers, and the EFS CSI
driver configuration including the PersistentVolume with the real filesystem
ID from the original EKS cluster.

## What I'd do differently

The JWT payload is empty — `jwt.sign({}, key)` — which means the token
proves authentication happened but carries no identity. When the tasks
service calls `/verify-token`, it cannot tell which user the token belongs
to, so task ownership checks silently fail. A real implementation would
embed at minimum a `userId` claim. I'd also move the credentials (MongoDB
connection string, JWT signing key) out of the manifest files and into
Kubernetes Secrets, and add liveness and readiness probes so EKS can
actually detect and restart unhealthy pods.
