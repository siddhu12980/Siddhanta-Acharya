Opinion Trading is a prediction market platform modeled after Probo — users
bet on real-world events by trading YES and NO outcome tokens, with prices
reflecting the market's probability estimate. The core is a custom
order-book matching engine written in Go, backed by a Redis queue for async
request routing and a WebSocket service for live orderbook feeds.

## What's interesting

The request flow is a synchronous RPC pattern built entirely on Redis. The
Express API gateway serializes each order into a JSON message, pushes it
onto the `"req"` list via `LPUSH`, then blocks waiting for a response on a
unique per-request pub/sub channel. The Go engine pops from `"req"` via
`BRPOP`, runs the matching logic in a worker goroutine, and publishes the
result back. This decouples the API layer from the engine completely — swap
Go for Rust tomorrow and nothing in the gateway changes.

The matching logic itself handles a subtle prediction market problem: what
do you do when there is no counterparty? Instead of leaving the order
unfilled, the engine creates an **inverse order** on the opposite side of
the book at the complementary price (YES@850 → inverse on NO@150). When
that inverse order matches later, both sides receive freshly minted tokens
rather than transferring existing stock. This means trading can happen
without any pre-existing supply — the market bootstraps its own liquidity.

The system originally started as a TypeScript engine (still in `engine/`)
before being rewritten in Go for better concurrency. Both implementations
exist in the repo side by side.

## What I'd do differently

The Go engine stores all state — order books, balances, positions — in
in-memory maps accessed by 10 goroutines with no mutex protection. This
is a race condition waiting to manifest under load. I'd add `sync.RWMutex`
per symbol, or restructure to a single-threaded engine with a channel-based
command queue. I'd also address the more fundamental gap: there is no event
resolution mechanism. The platform can trade positions but cannot settle
them — declaring a winner and distributing payouts was never built.
